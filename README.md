# NOVOLed B2B Каталог — Инструкция

> Проект работает без Supabase. Данные → Google Таблица → Apps Script → сайт на Vercel.

---

## ШАГ 1 — Настройка Google Таблицы

### 1.1 Структура листов

В вашей Google Таблице должно быть **два листа** с точными названиями:

#### Лист `products` — товары
| id | name | category | subcategory | series | socket | power | lumens | unit | price | in_stock | image_1 | image_2 | image_3 |
|---|---|---|---|---|---|---|---|---|---|---|---|---|---|
| LED-001 | Лампа H4 | LED Группа | H4 | D100 | H4 | 30 | 3000 | шт | 450 | TRUE | https://... | | |

- `id` — уникальный артикул (обязательно)
- `in_stock` — `TRUE` или `FALSE`
- `price` — число без символов валюты
- `image_1/2/3` — прямые ссылки на фото (Google Drive, Imgur и т.д.)

#### Лист `clients` — доступы клиентов
| email | active | can_see_prices |
|---|---|---|
| client@mail.ru | TRUE | TRUE |
| viewer@mail.ru | TRUE | FALSE |

- `email` — email клиента (строчными)
- `active` — `TRUE` = доступ есть, `FALSE` = заблокирован
- `can_see_prices` — `TRUE` = видит цены, `FALSE` = видит каталог без цен

## ШАГ 2 — Настройка Google Apps Script

### 2.1 Создать скрипт

1. Откройте вашу Google Таблицу
2. Меню → **Расширения → Apps Script**
3. Откроется редактор. Удалите весь код в файле `Code.gs`
4. Скопируйте содержимое файла `apps-script/Code.gs` из этого архива и вставьте
5. В начале файла найдите строку:
   ```
   const SPREADSHEET_ID = 'ВСТАВЬТЕ_ID_ТАБЛИЦЫ_СЮДА'
   ```
6. Замените на ID вашей таблицы (из URL таблицы: `docs.google.com/spreadsheets/d/**ВОТ_ЭТО**/edit`)

### 2.2 Опубликовать скрипт

1. Нажмите кнопку **«Развернуть»** (Deploy) → **«Новое развертывание»**
2. Тип: выберите **«Веб-приложение»**
3. Настройки:
   - Выполнять как: **Я** (ваш аккаунт)
   - Уровень доступа: **Все** (в т.ч. анонимные)
4. Нажмите **«Развернуть»**
5. **Скопируйте URL веб-приложения** — он вида `https://script.google.com/macros/s/ДЛИННЫЙ_КОД/exec`

> ⚠️ При первой публикации Google попросит выдать разрешения — нажмите «Разрешить».

### 2.3 Тест скрипта

Вставьте в браузер:
```
https://script.google.com/macros/s/ВАШ_КОД/exec?action=checkAccess&email=test@test.com
```
Должен вернуть JSON: `{"access":false,"can_see_prices":false}` — скрипт работает.

## КАК ДОБАВЛЯТЬ НОВЫХ КЛИЕНТОВ

Всё делается только в Google Таблице, лист **`clients`**:

1. Откройте таблицу → лист `clients`
2. В новой строке добавьте:
   - `email` — точный email клиента (строчными буквами!)
   - `active` — напишите `TRUE`
   - `can_see_prices` — `TRUE` если должен видеть цены, `FALSE` если только каталог
3. Сохранять ничего не нужно — Google Таблица сохраняет автоматически
4. Клиент может войти **немедленно** — никаких пересборок сайта не нужно

### Заблокировать клиента:
Найдите его строку в `clients`, измените `active` на `FALSE`.
Клиент при следующей загрузке страницы потеряет доступ.

### Убрать доступ к ценам:
Измените `can_see_prices` на `FALSE`. Цены исчезнут при следующей загрузке.

## КАК ОБНОВЛЯТЬ ТОВАРЫ

1. Откройте Google Таблицу → лист `products`
2. Редактируйте, добавляйте, удаляйте строки
3. Изменения появятся на сайте **автоматически** — каталог читается напрямую из таблицы

### Автообновление:
- GitHub Actions каждые **2 часа** делает коммит `.last-sync` → Vercel пересобирает сайт
- Клиент нажимает **«Обновить каталог»** на сайте — данные обновляются мгновенно
- Кэш браузера на данные — **5 минут**, потом автоматически перечитывает из таблицы

### Принудительное обновление вручную через GitHub:
Actions → **Auto Refresh Catalog** → **Run workflow**

## ВХОД ДЛЯ КЛИЕНТОВ

Авторизация теперь **без пароля** — только email:
1. Клиент заходит на `/login.html`
2. Вводит свой email
3. Apps Script проверяет его в таблице `clients`
4. Если `active = TRUE` — вход выполнен, загружается каталог

> Если клиент хочет иметь пароль — это можно добавить позже (добавить колонку `password` в таблицу clients и проверку в Code.gs).

## ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ

### Каталог пустой
- Проверьте URL в `js/api.js` — он должен быть реальным URL скрипта
- Откройте консоль браузера (F12) → вкладка Network — смотрите запрос к `script.google.com`
- Убедитесь что скрипт опубликован с доступом «Все»

### Вход не работает
- Проверьте URL в `js/auth.js`
- Проверьте что email в таблице `clients` написан строчными буквами
- Проверьте что колонки называются точно `email`, `active`, `can_see_prices`

### Скрипт вернул ошибку про SPREADSHEET_ID
- Убедитесь что вставили правильный ID таблицы в `Code.gs`
- После изменения Code.gs нужно сделать **новое развертывание** (не редактировать старое!)
